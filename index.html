Procedure planning from visual endpoints

Masked Temporal Interpolation Diffusion for Procedure Planning in Instructional Videos

From start and goal observations, MTID interpolates latent mid-state features and injects them into a masked denoising process to recover the missing action sequence.

Why MTID?

Seeing the states between the states.

TL;DRMTID replaces coarse text-only guidance with temporally interpolated visual features, enabling coherent and task-aligned procedure planning.

Procedure planning infers a sequence of actions that transforms an observed start state into a desired goal state. Existing approaches largely rely on task labels, event descriptions, or knowledge graphs; these signals cannot fully describe the visual evolution hidden between the endpoints.

MTID learns that evolution directly in latent space. Its intermediate visual features supervise an end-to-end diffusion planner, while an action-aware mask and task-adaptive proximity objective suppress irrelevant predictions.

Method

Three components, one end-to-end planner.

MTID decomposes procedure planning into task recognition, latent mid-state reconstruction, and conditional action generation.

01

Task supervision

A transformer classifier reads start and goal observations and predicts the task class that constrains planning.

02

Temporal interpolation

A learnable interpolation matrix reconstructs intermediate latent visual features as mid-state supervision.

03

Masked diffusion

Action-aware projection narrows the search space while DDIM recovers a coherent action sequence efficiently.

Core module

Latent space temporal interpolation

The observation encoder maps the endpoints to latent features. A learnable coefficient generates intermediate representations, which transformer blocks refine into temporally consistent visual supervision.

  • Task-adaptive interpolation ratio
  • Transformer refinement with positional encoding
  • Cross-attention injection into the U-Net planner
